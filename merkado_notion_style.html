This document covers the full typography and theming strategy for the Grascope Product Suite, built on the Merkado OS shared design system. Four apps share one package — each with a distinct visual identity configured through a minimal config object.

🏛 Architecture Decision

One shared package (merkado_ds) holds all tokens, font files, theme builders, and Mix 2.0 styles. Each app fills in a single config object — that's the entire theme footprint per app.

💡
Why one package? Typography and Spacing semantics are identical across all apps. Only color semantics differ. A shared package prevents copy-paste drift — no more one app with borderRadius: 12 and another with borderRadius: 14 six months later.

🔤 Typography Strategy

Each product serves a different emotional and functional context. Typography reflects this while maintaining enough consistency for users to recognize the ecosystem. All shared UI components — buttons, forms, navigation — use Inter across all four apps.

⚠️
SF Pro Note (MyCut): SF Pro appears in the Figma as MyCut's font. It is Apple's system font and cannot be bundled legally. The typography doc already resolves this — Inter achieves the same professional, neutral feel and is used by Stripe, GitHub, and Linear. Confirm with your designer this was a Figma placeholder.

Font personality rationale

▶ ✂️ MyCut — Inter

Trustworthy, professional, no-nonsense. This is about money and contracts — not a lifestyle app.

  • Designed for UI readability from 12px upward
  • Excellent tabular figures for money amounts, deal IDs, dates
  • 9 weights — full hierarchy from display to caption
  • DM Mono secondary for deal IDs (MYCUT-12345), wallet addresses, transaction references
▶ 👗 Haulway — DM Sans

Trendy, energetic, aspirational. Social-first, visual-heavy, fast-paced — evokes TikTok, Instagram, Pinterest.

  • Modern geometric, slightly rounded — friendlier than Inter
  • Designed for low x-height mobile screens
  • Clean at small sizes for product tags on video
  • Outfit secondary for hero sections and creator profile headers at 24px+
▶ 🪡 Driply — Outfit

Precise, crafted, premium quality. Evokes fabric labels, measurement charts, bespoke tailoring.

  • Geometric, clean, modern — evokes precision like measuring tape increments
  • Good for specifications, measurements, sizes, fabric types
  • EB Garamond secondary for premium/luxury contexts — bridal gown commissions, verified designer profiles only
▶ 🍜 FeastFeed — Manrope

Appetizing, local, authentic. Celebrates street food, home cooking, cultural food.

  • Geometric with slight warmth — rounded corners, modern but not cold
  • 7 weights including ExtraBold — strong vendor names, bold dish names
  • Lobster secondary for the logo and category headers only. Script/handwritten feel evokes food trucks and chalkboard menus. Never for body text. Never small.

⚙️ Setup Guide

Downloading the Fonts

All fonts are on Google Fonts — free and open-source. Download each, unzip, and place the .ttf files inside merkado_ds/fonts/. Never put font files inside individual app directories.

📎
If the zip contains a variable font (e.g. DMSans[opsz,wght].ttf), use it instead — one file covers all weights and simplifies your pubspec declarations.

Per-App pubspecs

Each app re-declares only its own fonts, referencing the package path. Flutter doesn't inherit font assets from packages automatically — one-time copy per app. Each app only pulls in what it uses.

⚠️
Driply does not declare Lobster. MyCut does not declare Manrope. Declare only what your app actually renders — keep bundle sizes lean.

🎨 Theme System

Font Family Constants

Never hardcode font name strings anywhere in the codebase. Change a font in one place and it propagates everywhere.

dart — font_families.dart
class MerkadoFonts { MerkadoFonts._(); // Shared UI — all buttons, forms, nav across every app static const String ui = 'Inter'; static const String mono = 'DM Mono'; // Product primary fonts — pass to fontFamilyOverride static const String mycut = 'Inter'; // same as UI, explicit static const String haulway = 'DM Sans'; static const String feastfeed = 'Manrope'; static const String driply = 'Outfit'; // Secondary / display — used in widgets, NOT in ThemeData static const String haulwayDisplay = 'Outfit'; // hero 24px+ static const String feastfeedDisplay = 'Lobster'; // logo only static const String driplyPremium = 'EB Garamond'; // bridal static const String mycutMono = 'DM Mono'; // IDs, refs }

AppColorScheme

The normalized color contract. The theme builder only ever reads from this class — it never imports MycutLightColors or DriplyDarkColors directly. That layer of indirection is what makes the builder reusable.

Each app's semantic color file provides a factory method that maps its tokens into the shared contract.
dart — app_color_scheme.dart
class AppColorScheme { final Color brandPrimary; final Color brandSecondary; final Color backgroundPrimary; final Color backgroundSurface; final Color textPrimary; final Color textSecondary; final Color textDisabled; final Color borderDefault; final Color borderFocused; final Color stateSuccess; final Color stateWarning; final Color stateError; final Color surfaceContainer; final Color surfaceCard; // One factory per app factory AppColorScheme.fromMycut({required bool isDark}) {...} factory AppColorScheme.fromHaulway({required bool isDark}) {...} factory AppColorScheme.fromDriply({required bool isDark}) {...} factory AppColorScheme.fromFeastfeed({required bool isDark}) {...} }

MerkadoThemeConfig + Builder

Config is what each app fills in. Builder assembles the ThemeData. Every sub-theme is a modular private function — pass an override in the config to replace any slot, or leave it null to use the package default.

dart — merkado_theme_config.dart
class MerkadoThemeConfig { final AppColorScheme colors; final String? fontFamilyOverride; // use MerkadoFonts.xxx final double borderRadius; // Sub-theme overrides — null = use package default final AppBarTheme? appBarThemeOverride; final ElevatedButtonThemeData? elevatedButtonOverride; final InputDecorationTheme? inputDecorationOverride; final BottomSheetThemeData? bottomSheetOverride; final ChipThemeData? chipOverride; final CheckboxThemeData? checkboxOverride; const MerkadoThemeConfig({ required this.colors, this.fontFamilyOverride, this.borderRadius = 12.0, // all overrides default to null }); }

📱 App Theme Files

This is the entire theme footprint each app writes. One import, fill in the config, wire into MaterialApp.

dart — haulway_theme.dart (example)
import 'package:merkado_ds/merkado_ds.dart'; class HaulwayTheme { HaulwayTheme._(); static ThemeData get light => MerkadoThemeBuilder.build( isDark: false, config: MerkadoThemeConfig( colors: AppColorScheme.fromHaulway(isDark: false), fontFamilyOverride: MerkadoFonts.haulway, borderRadius: 12.0, ), ); static ThemeData get dark => MerkadoThemeBuilder.build( isDark: true, config: MerkadoThemeConfig( colors: AppColorScheme.fromHaulway(isDark: true), fontFamilyOverride: MerkadoFonts.haulway, borderRadius: 12.0, ), ); }
dart — main.dart
MaterialApp( theme: HaulwayTheme.light, darkTheme: HaulwayTheme.dark, themeMode: ThemeMode.system, // Mix 2.0 token provider — wraps entire widget tree builder: (context, child) => MixTokenProvider( colors: AppColorScheme.fromHaulway( isDark: Theme.of(context).brightness == Brightness.dark, ), child: child!, ), )
🎉
Adding a new app (e.g. ItsYourDay) means: add its Figma-generated color files, write one AppColorScheme.fromItsYourDay() factory, and a 15-line theme file. Nothing else in merkado_ds changes.
