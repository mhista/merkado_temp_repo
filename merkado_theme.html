01 — Overview

Architecture Decision

One shared package merkado_ds holds all tokens, fonts, theme builders, and Mix styles. Each app fills in one config object — that's its entire theme footprint.

💡
Typography and Spacing semantics are identical across all apps. Only color semantics differ per app. The theme builder never imports app-specific color classes directly — it reads through AppColorScheme.

05 — Apps

Per-App pubspec.yaml

Each app re-declares only its own fonts, referencing the package path. Flutter doesn't inherit font assets from packages automatically — one-time setup per app.

Each app declares only the fonts it actually uses. Driply doesn't pull in Lobster. MyCut doesn't pull in Manrope. Keep bundles lean.

06 — Theme System

Font Family Constants

Never hardcode font name strings anywhere. One file, one source of truth. Change a font in one place.

merkado_ds/lib/src/tokens/font_families.dartdart
class MerkadoFonts { MerkadoFonts._(); // Shared UI — buttons, forms, nav across all apps static const String ui = 'Inter'; static const String mono = 'DM Mono'; // Product primary fonts — pass to fontFamilyOverride static const String mycut = 'Inter'; // same as UI, explicit static const String haulway = 'DM Sans'; static const String feastfeed = 'Manrope'; static const String driply = 'Outfit'; // Secondary / display — used in specific widgets only, not ThemeData static const String haulwayDisplay = 'Outfit'; // hero / creator 24px+ static const String feastfeedDisplay = 'Lobster'; // logo / category headers static const String driplyPremium = 'EB Garamond'; // bridal / high-end static const String mycutMono = 'DM Mono'; // deal IDs, tx refs }
07 — Theme System

AppColorScheme

The normalized color contract. The theme builder reads only this — it never imports app-specific classes like MycutLightColors directly. That's the whole point.

merkado_ds/lib/src/tokens/semantic/colors/app_color_scheme.dartdart
class AppColorScheme { final Color brandPrimary; final Color brandSecondary; final Color backgroundPrimary; final Color backgroundSurface; final Color textPrimary; final Color textSecondary; final Color textDisabled; final Color borderDefault; final Color borderFocused; final Color stateSuccess; final Color stateWarning; final Color stateError; final Color surfaceContainer; final Color surfaceCard; const AppColorScheme({ required this.brandPrimary, this.brandSecondary = const Color(0xFF000000), required this.backgroundPrimary, // ... all other required fields }); // One factory per app — maps semantic tokens into this contract factory AppColorScheme.fromMycut({required bool isDark}) { /* ... */ } factory AppColorScheme.fromHaulway({required bool isDark}) { /* ... */ } factory AppColorScheme.fromDriply({required bool isDark}) { /* ... */ } factory AppColorScheme.fromFeastfeed({required bool isDark}) { /* ... */ } }
08 — Theme System

MerkadoThemeConfig + Builder

Config is what each app fills in. Builder assembles ThemeData. Pass an override in config to replace any sub-theme — null means use the default.

merkado_ds/lib/src/theme/merkado_theme_config.dartdart
class MerkadoThemeConfig { final AppColorScheme colors; final String? fontFamilyOverride; final double borderRadius; // Sub-theme overrides — null = use package default final AppBarTheme? appBarThemeOverride; final ElevatedButtonThemeData? elevatedButtonOverride; final InputDecorationTheme? inputDecorationOverride; final BottomSheetThemeData? bottomSheetOverride; final ChipThemeData? chipOverride; final CheckboxThemeData? checkboxOverride; const MerkadoThemeConfig({ required this.colors, this.fontFamilyOverride, this.borderRadius = 12.0, this.appBarThemeOverride, this.elevatedButtonOverride, this.inputDecorationOverride, this.bottomSheetOverride, this.chipOverride, this.checkboxOverride, }); }

10 — Done

Each App's Theme File

One import. Fill in the config. This is the entire theme footprint per app.

haulway/lib/theme/haulway_theme.dartdart
import 'package:merkado_ds/merkado_ds.dart'; class HaulwayTheme { HaulwayTheme._(); static ThemeData get light => MerkadoThemeBuilder.build( isDark: false, config: MerkadoThemeConfig( colors: AppColorScheme.fromHaulway(isDark: false), fontFamilyOverride: MerkadoFonts.haulway, borderRadius: 12.0, ), ); static ThemeData get dark => MerkadoThemeBuilder.build( isDark: true, config: MerkadoThemeConfig( colors: AppColorScheme.fromHaulway(isDark: true), fontFamilyOverride: MerkadoFonts.haulway, borderRadius: 12.0, ), ); }
any_app/lib/main.dart — wiring it updart
MaterialApp( theme: HaulwayTheme.light, darkTheme: HaulwayTheme.dark, themeMode: ThemeMode.system, // Mix 2.0 token provider — wraps entire tree builder: (context, child) => MixTokenProvider( colors: AppColorScheme.fromHaulway( isDark: Theme.of(context).brightness == Brightness.dark, ), child: child!, ), )
🎉
Adding a new app (e.g. ItsYourDay) means: add its Figma-generated color files, write one AppColorScheme.fromItsYourDay() factory, and a 15-line theme file. Nothing else in the package changes.
